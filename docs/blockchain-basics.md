# ブロックチェーン基礎概念

## スマートコントラクトとは

普通のWebアプリでは、ロジックは**サーバー**に、データは**データベース**に保存する。

ブロックチェーンでは、その**サーバー + データベースの役割**を「スマートコントラクト」が担う。

```
従来のWebアプリ:
  ユーザー → サーバー（ロジック） → データベース（保存）

ブロックチェーンアプリ:
  ユーザー → スマートコントラクト（ロジック + 保存）
```

スマートコントラクトは：
- **Solidity** という言語で書く（JavaScriptに似てる）
- ブロックチェーン上に**デプロイ（公開）** すると、世界中の誰でもアクセスできる
- 一度デプロイしたら**改ざん不可能** — コードもデータも透明
- 関数を呼び出すことでデータの読み書きをする

簡単な例：

```solidity
// これがスマートコントラクト。クラスに近い
contract VoteGame {
    // データベースのカラムのようなもの
    uint public totalVotes;

    // APIエンドポイントのようなもの
    function vote() public {
        totalVotes += 1;
    }
}
```

## Foundry — コントラクトの開発ツール

Foundry は、Next.js における**Node.js + npm** のような存在。

| Foundry のツール | 役割 | Web開発で例えると |
|---|---|---|
| `forge build` | コントラクトをコンパイル | `npm run build` |
| `forge test` | テスト実行 | `npm test` |
| `forge script` | デプロイ（公開） | Vercelにデプロイ |
| `anvil` | ローカルのブロックチェーン | `localhost:3000` の開発サーバー |

つまり、Solidity を書いて、Foundry でビルド・テスト・デプロイする流れ。

## viem — フロントエンドからコントラクトを呼ぶライブラリ

viem は、フロントエンド（Next.js）からスマートコントラクトの関数を呼び出すためのライブラリ。

```
Web開発で例えると:
  fetch("/api/vote", { method: "POST" })   ← APIを呼ぶ

ブロックチェーンでは:
  viem で contract.vote() を呼ぶ            ← コントラクトの関数を呼ぶ
```

```typescript
// イメージ（実際のコードに近い）
import { createPublicClient } from "viem";

// コントラクトからデータを読む（無料）
const totalVotes = await client.readContract({
  address: "0x1234...",    // コントラクトのアドレス（URLのようなもの）
  functionName: "totalVotes",
});

// コントラクトにデータを書く（ガス代が必要）
await walletClient.writeContract({
  address: "0x1234...",
  functionName: "vote",
});
```

重要なポイント：
- **読み取り（read）** → 無料、即座
- **書き込み（write）** → ガス代（テストネットではタダ）が必要、数秒かかる

## Privy — ユーザー認証 + ウォレット管理

ブロックチェーンに書き込むには「ウォレット」が必要。ウォレット = ブロックチェーン上の**ユーザーアカウント + 署名機能**。

通常、ユーザーは MetaMask などのウォレットアプリを自分でインストールする必要があるが、**Privy を使うとそれが不要**になる。

```
MetaMask の場合:
  ユーザーが自分でインストール → ウォレット作成 → 接続 → 署名

Privy の場合:
  メールかGoogleでログイン → Privy が裏でウォレットを自動作成 → 完了
```

ユーザーから見ると**普通のログインと同じ体験**で、裏側ではブロックチェーン操作に必要なウォレットが自動で用意される。

---

## 全体の流れ（このプロジェクト）

```
1. ユーザーがメールでログイン（Privy）
       ↓
2. Privy が裏でウォレットを作成
       ↓
3. ユーザーが「上」ボタンをクリック
       ↓
4. viem がスマートコントラクトの vote("上") を呼び出す
       ↓
5. ブロックチェーンに投票が記録される
       ↓
6. 時間終了後、コントラクトから投票データを読み取り
       ↓
7. フロントエンドでキャラ移動アニメーション表示
       ↓
8. 宝箱到達 → コントラクトが報酬を配布
```

## オンチェーン vs オフチェーン

### オンチェーン（ブロックチェーン上）

データや処理を**ブロックチェーンに直接記録**する。

- 全世界に公開・改ざん不可能
- **毎回ガス代（手数料）がかかる**
- 処理速度が遅い（数秒〜数分）
- スマートコントラクトで実行

### オフチェーン（ブロックチェーン外）

データや処理を**通常のサーバー・DBに保存**する。従来のWebアプリと同じ。

- 自分のサーバーだけに保存
- **無料・高速**
- 改ざんは技術的に可能（自分が管理するため）

## contracts/ フォルダの構造

```
contracts/
├── src/           # コントラクト本体（プロダクションコード）
├── test/          # テスト
├── script/        # デプロイスクリプト
├── lib/           # 外部ライブラリ（node_modules のようなもの）
└── foundry.toml   # 設定ファイル（next.config.ts のようなもの）
```

| フォルダ | 役割 | Web開発で例えると |
|---|---|---|
| `src/` | コントラクトのコード | `src/app/api/` (APIルート) |
| `test/` | テスト | `__tests__/` |
| `script/` | デプロイ手順 | CI/CDのデプロイ設定 |
| `lib/` | 外部ライブラリ | `node_modules/` |

## Solidity の基本構文（TypeScript との対応）

| Solidity | TypeScript で例えると | 意味 |
|---|---|---|
| `enum Direction { Up, Down, Left, Right }` | `enum Direction { Up, Down, Left, Right }` | ほぼ同じ。内部的に 0,1,2,3 の数値 |
| `struct Vote { ... }` | `interface Vote { ... }` | データの型定義 |
| `Vote[] public votes` | `const votes: Vote[] = []` | 配列。`public` にすると外から読める |
| `external` | `export` 的 | 外部から呼べる関数 |
| `view` | — | 読み取り専用（ガス代無料） |
| `memory` | — | 戻り値を一時メモリに載せて返す |
| `msg.sender` | `req.user.id` 的 | 呼び出した人のアドレス（自動） |
| `block.timestamp` | `Date.now()` | 現在時刻（自動） |
